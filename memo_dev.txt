🎬 具体的な開発の流れ（初心者向け）

  Phase 1: 準備（初回のみ、30分）

  # 1. EAS CLIインストール
  npm install -g eas-cli

  # 2. Expoアカウントでログイン
  eas login

  # 3. プロジェクトの設定
  eas build:configure

  Phase 2: Dev Clientビルド（初回のみ、15-30分）

  # 1. expo-dev-clientをインストール
  npx expo install expo-dev-client

  # 2. Dev Clientをビルド（クラウドで自動ビルド）
  eas build --profile development --platform ios

  # ビルド中は待つだけ（15-30分）
  # 完了したらQRコードが表示される

  Phase 3: Dev Clientをインストール（初回のみ、5分）

  # iOSシミュレータの場合
  # ビルド完了後に表示されるURLからダウンロード
  # .appファイルをシミュレータにドラッグ&ドロップ

  # 実機の場合
  # TestFlightまたはQRコードからインストール

  Phase 4: 開発開始（毎回、これが日常）

  # 1. 開発サーバー起動
  npx expo start --dev-client

  # 2. Dev Clientアプリを開く（実機またはシミュレータ）

  # 3. QRコードスキャンまたは「Enter URL manually」

  # 4. 開発開始！
  # → コード変更は Fast Refresh で即座に反映
  # → バックグラウンド再生も動く ✅

  ---
  🆚 Expo Go vs Dev Client（日常使用）

  Expo Go（今まで）

  1. npx expo start --tunnel
  2. Expo Goアプリ起動
  3. QRスキャン
  4. 開発

  利点: すぐ始められる
  欠点: バックグラウンド再生できない、広告動かない

  Dev Client（これから）

  1. npx expo start --dev-client
  2. Dev Clientアプリ起動
  3. QRスキャン
  4. 開発

  利点: バックグラウンド再生できる、広告動く
  欠点: 初回ビルドに15-30分かかる（1回だけ）

  日常の開発速度: Expo Go と同じ！

  ---
  💡 重要なポイント

  ✅ 良いニュース

  1. Dev Clientは1回ビルドすれば、あとはExpo Goと同じように使える
  2. Fast Refreshも動く（コード変更が即座に反映）
  3. 追加のネイティブコードは不要

  ⚠️ 注意点

  1. 初回ビルドに15-30分かかる（コーヒー休憩）
  2. ネイティブ依存関係を変更したら再ビルド必要
    - 例: 新しいネイティブパッケージを追加
    - 例: app.json の設定を変更
  3. JavaScriptコードの変更は再ビルド不要
